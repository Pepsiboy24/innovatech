🔧 Form Validation Fix Test

This page tests the fix for the "invalid form control is not focusable" error.

🐛 Problem Identified

ISSUE: Required fields in hidden steps caused validation errors

The browser tried to validate all required fields including those in hidden steps, but couldn't focus on invisible elements.

✅ Solution Applied

FIXED: Dynamic required attribute management

🔧 Technical Fix Details

1. Removed Static Required Attributes

<input type="date" id="startDate" name="startDate" required> <input type="date" id="startDate" name="startDate">

2. Added Dynamic Required Management

function manageRequiredAttributes(step) { // Remove required from ALL fields first const allRequiredFields = document.querySelectorAll('[required]'); allRequiredFields.forEach(field => { field.removeAttribute('required'); }); // Add required only to CURRENT step fields const currentStepEl = document.getElementById("step" + step); if (currentStepEl) { const stepFields = currentStepEl.querySelectorAll('input, select, textarea'); stepFields.forEach(field => { const label = field.closest('.form-group')?.querySelector('label'); if (label && label.innerHTML.includes('class="required"')) { field.setAttribute('required', ''); } }); } }

3. Enhanced Validation

// Added checkbox validation for Step 3 if (step === 3) { const subjectsChecked = step3El.querySelectorAll('input[name="subjects"]:checked').length > 0; const gradeLevelsChecked = step3El.querySelectorAll('input[name="gradeLevels"]:checked').length > 0; if (!subjectsChecked) showToast("Please select at least one certified subject.", "warning"); if (!gradeLevelsChecked) showToast("Please select at least one grade level.", "warning"); }
